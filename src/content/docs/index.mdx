## Canonical Repositories

- [`HaloForgeAI/HaloForge`](https://github.com/HaloForgeAI/HaloForge) contains the desktop runtime, plugin loader, and local app shell.
- [`HaloForgeAI/haloforge-plugin-api`](https://github.com/HaloForgeAI/haloforge-plugin-api) contains the Rust API crate, TypeScript SDK, packager, templates, and plugin authoring skill.
- [`HaloForgeAI/HaloForge-Community-Cloud`](https://github.com/HaloForgeAI/HaloForge-Community-Cloud) contains public Cloudflare Workers and Pages deployment surfaces.
- [`HaloForgeAI/HaloForge-Enterprise-Server`](https://github.com/HaloForgeAI/HaloForge-Enterprise-Server) contains enterprise deployment and private control plane code.

## Current Focus

The first complete documentation block is plugin development. The next blocks should cover Agent Orchestra external providers, catalog publishing, enterprise server integration, Streamable HTTP MCP, and knowledge base APIs.

## Enterprise Server

- Start here if you are trying to open a local Enterprise deployment for the first time: [Enterprise Server](/enterprise/).
- The main operator entry is typically `http://127.0.0.1:8788/admin`, not the root `/` page.
- The browser KB reader is typically `http://127.0.0.1:8788/kb`.
