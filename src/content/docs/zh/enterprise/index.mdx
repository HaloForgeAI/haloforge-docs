HaloForge Enterprise Server 本质上是一个**企业控制面**，不是 HaloForge 桌面端本体。

它当前提供的是企业 API、管理后台、知识库阅读器、私有 catalog、知识导入与检索流程、模型网关管理以及 MCP bridge。如果你打开根路径只看到一个简单落地页，并不代表服务没起来；那一页只是运维入口壳。

## 真实入口

| 界面 | 本地常见地址 | 作用 |
| --- | --- | --- |
| 服务落地页 | `http://127.0.0.1:8788/` | 说明服务已运行，并给出真实入口链接 |
| 管理后台 | `http://127.0.0.1:8788/admin` | 企业管理员主工作台 |
| 知识库阅读器 | `http://127.0.0.1:8788/kb` | 同域知识阅读与搜索界面 |
| MCP bridge | `http://127.0.0.1:8790/mcp` | 面向工具和 AI 客户端的 Streamable HTTP MCP 入口 |
| 健康检查 | `http://127.0.0.1:8788/health` | JSON 健康探针 |

<CardGrid>
  <Card title="当前偏管理后台" icon="setting">
    现在的产品重点是运营和管理，所以真正主入口是 `/admin`，不是一个对外展示型首页。

  <Card title="已经有知识阅读器" icon="book">

  </Card>
</CardGrid>

## 这个仓库当前负责什么

- 企业认证和后台会话
- 租户、组织与 ACL 治理
- 私有 catalog 和平台资产管理
- 知识源、文档导入、KB 页面以及 RAG 检索链路
- 模型网关、额度和请求日志
- 面向 KB 的企业 MCP 能力

## 一个重要预期

这个仓库现在交付的是企业控制面和知识运营底座，并不是一个从根路径打开就很像“完整企业门户”的成品网站。如果你希望有更强的第一印象、引导式登录和更美观的首页，那应该作为单独的产品设计与前端改造任务继续推进。
