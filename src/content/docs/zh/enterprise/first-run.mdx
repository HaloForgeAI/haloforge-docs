第一次使用时最容易误判的点其实很简单：

- `http://127.0.0.1:8788/` **不是**主工作台
- `http://127.0.0.1:8788/admin` 才是实际管理后台入口
- 默认 Compose 本地栈如果没有启用任何登录方式，看起来就会像“页面是死的”

## 推荐本地配置

本地验证时，建议先在 `HaloForge-Enterprise-Server/deploy/.env` 里打开一个可用登录路径，然后再启动或重启服务。

推荐本地参数：

```bash
HF_ENTERPRISE_ALLOW_MOCK_AUTH=true
HF_ENTERPRISE_EMAIL_AUTH_ENABLED=true
HF_ENTERPRISE_SEED_DATA=true
```

这些参数分别意味着：

- `HF_ENTERPRISE_ALLOW_MOCK_AUTH=true`：`/admin` 会出现一键 `Mock admin` 登录按钮
- `HF_ENTERPRISE_EMAIL_AUTH_ENABLED=true`：启用邮箱验证码登录表单
- 当邮箱登录开启且 mock auth 也开启时，即使没有真实 SMTP，本地界面也可以直接显示预览验证码
- `HF_ENTERPRISE_SEED_DATA=true`：本地探索时非常有用，否则后台空空的，体验上很像系统坏了

## 本地打开步骤

<Steps>
  1. 启动服务，并确认 `http://127.0.0.1:8788/health` 能返回正常 JSON。

  2. 打开 `http://127.0.0.1:8788/admin`。

  3. 选择一个已经启用的登录方式：

     - `Mock admin`：本地最快验证路径
     - `Email`：验证码登录测试
     - `OIDC` 或 `WeCom`：真实企业身份源联调

  4. 登录后先看左侧导航：

     - `总览`：健康状态、数量统计、部署姿态
     - `知识运营`：连接器、同步任务、知识源、MCP Token
     - `功能中心`：私有 catalog 和上传中心
     - `网关`：模型密钥、通道和流量日志
     - `任务`：Agent 会话和消息保留

  5. 保持同一个浏览器会话，再打开 `http://127.0.0.1:8788/kb` 验证知识库阅读器。
</Steps>

## 如果你看到的只是一个简单静态页

通常是以下几种情况之一：

- 你打开的是 `/` 而不是 `/admin`
- 服务已经健康，但没有启用任何登录方式
- 系统里没有 seed 数据，所以虽然能进，但内容几乎为空

## 更适合生产的登录方式

生产环境建议使用：

- `OIDC`
- `WeCom`
- `Email OTP`

`Mock admin` 只适合本地开发和联调，不应该作为生产登录方案。
