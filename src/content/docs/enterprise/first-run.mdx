The most common first-run confusion is simple:

- `http://127.0.0.1:8788/` is **not** the main workspace
- `http://127.0.0.1:8788/admin` is the actual admin entry
- a fresh local Compose file can start with every login button disabled unless you enable a local auth path

## Recommended Local Setup

For local validation, enable a login method in `HaloForge-Enterprise-Server/deploy/.env` before starting or restarting the stack.

Recommended local flags:

```bash
HF_ENTERPRISE_ALLOW_MOCK_AUTH=true
HF_ENTERPRISE_EMAIL_AUTH_ENABLED=true
HF_ENTERPRISE_SEED_DATA=true
```

What each flag does:

- `HF_ENTERPRISE_ALLOW_MOCK_AUTH=true` enables the one-click `Mock admin` button on `/admin`
- `HF_ENTERPRISE_EMAIL_AUTH_ENABLED=true` enables the email OTP form
- when email auth is enabled and mock auth is also enabled, local login can show a preview verification code directly in the UI even without a real SMTP service
- `HF_ENTERPRISE_SEED_DATA=true` is helpful for local exploration because an empty control plane feels broken even when it is technically healthy

## Local Browser Checklist

<Steps>
  1. Start the deployment and wait for `http://127.0.0.1:8788/health` to return OK JSON.

  2. Open `http://127.0.0.1:8788/admin`.

  3. Pick one of the enabled login methods:

     - `Mock admin` for the fastest local validation path
     - `Email` for OTP-style local testing
     - `OIDC` or `WeCom` for real enterprise identity provider testing

  4. After sign-in, use the left navigation:

     - `总览` for health, counts, and readiness
     - `知识运营` for connectors, sync jobs, knowledge sources, and MCP token management
     - `功能中心` for private catalog and uploads
     - `网关` for model keys, channels, and traffic
     - `任务` for agent session retention and event history

  5. Open `http://127.0.0.1:8788/kb` in the same browser session to verify the KB reader.
</Steps>

## If You Only See A Simple Static Page

That usually means one of these is true:

- you opened `/` instead of `/admin`
- the service is healthy, but no login method is enabled
- the system is running with no seeded content, so the admin data feels empty

## Production-Oriented Login Modes

Use these for real deployments:

- `OIDC` for enterprise SSO
- `WeCom` for enterprise WeCom QR sign-in
- `Email OTP` for smaller managed setups

`Mock admin` is a local-development convenience and should not be the target production path.
