HaloForge Enterprise Server is a **private control plane**, not the HaloForge desktop app itself.

It currently ships the enterprise API, admin console, KB reader, private catalog surfaces, knowledge ingestion workflows, model gateway administration, and MCP bridge. If you open the root URL and only see a simple landing page, the service is still running correctly. That root page is only an operational shell.

## Real Entry Points

| Surface | Typical local URL | Purpose |
| --- | --- | --- |
| Service landing | `http://127.0.0.1:8788/` | Confirms the server is running and links to the real surfaces |
| Admin console | `http://127.0.0.1:8788/admin` | Main operator workspace for enterprise admins |
| KB reader | `http://127.0.0.1:8788/kb` | Same-origin knowledge reading and search experience |
| MCP bridge | `http://127.0.0.1:8790/mcp` | Streamable HTTP MCP endpoint for tools and AI clients |
| Health | `http://127.0.0.1:8788/health` | JSON health probe |

## What This Repository Owns

- Enterprise authentication and admin sessions
- Tenant, organization, and ACL governance
- Private catalog and platform asset management
- Knowledge sources, document import, KB pages, and RAG-backed retrieval
- Model gateway administration, quotas, and request logs
- Enterprise MCP access to KB search and reading

## Important Expectation

This repository currently provides the enterprise control plane and knowledge operations foundation. It does **not** yet present itself like a finished employee-facing portal at the root URL. If you need a polished first-impression web experience, that should be treated as a separate product-design and frontend task on top of the existing admin and KB surfaces.
